# QuickSum AI — Backend (نسخة حقيقية شغالة)

ده backend حقيقي وشغال بالفعل: تسجيل حساب حقيقي، تسجيل دخول، قاعدة بيانات، رفع صورة فاتورة، قراءة أرقامها بالذكاء الاصطناعي عبر Anthropic API، تأكيد البريد الإلكتروني، نسيت كلمة المرور، وحماية أمنية أساسية (Rate limiting + Helmet). تم اختباره بالكامل قبل التسليم.

## تشغيله محليًا

```bash
npm install
cp .env.example .env
# افتح .env وحط قيمة ANTHROPIC_API_KEY (من https://console.anthropic.com)
npm start
```

افتح `http://localhost:3000` — هتلاقي واجهة فيها تسجيل دخول/حساب جديد شغالة بجد.

**من غير ANTHROPIC_API_KEY**: التطبيق لسه شغال بالكامل (حساب، حفظ، سجل، إحصائيات) لكن قراءة الأرقام من الصورة هترجع بيانات فاضية مع تحذير واضح بدل ما تفشل السيرفر.

**من غير إعدادات SMTP**: روابط تأكيد البريد وإعادة تعيين كلمة المرور هتظهر في استجابة الـ API نفسها (وفي الـ console) بدل ما تتبعت بريد فعلي — كده تقدر تجرب الميزتين بالكامل من غير ما تحتاج حساب بريد فعلي وقت التطوير. لازم تضبط SMTP قبل الإطلاق الحقيقي.

## وضع "طلب انضمام" (Invite-only) — للإطلاق التجريبي المغلق

لو عايز تتحكم مين يستخدم التطبيق بدل التسجيل المفتوح للجميع (مفيد جدًا لبداية تجريبية قبل أي تسجيل رسمي/دفع):

1. حط في `.env`: `INVITE_ONLY=true` و`ADMIN_KEY=` (اختار نص عشوائي طويل)
2. التسجيل المباشر (`/api/auth/register`) هيتقفل تلقائيًا، وهيظهر في الواجهة تبويب **"طلب انضمام"** بدل "حساب جديد"
3. أي حد يقدّم طلب (اسمه، بريده، هاتفه، اسم نشاطه، رسالة) عبر الموقع
4. افتح `/admin` (مثلاً `http://localhost:3000/admin`)، حط الـ `ADMIN_KEY`، وهتشوف الطلبات
5. لما توافق على طلب، بيتعمل حساب تلقائي وبيتبعت للشخص رابط "عيّن كلمة مرورك" (بريد فعلي لو SMTP مضبوط، أو رابط تجريبي في الـ response/console لو لسه من غير SMTP)

كده تقدر تشغّل نسخة تجريبية مغلقة لعدد محدود من التجار الحقيقيين من غير أي حاجة رسمية (سجل تجاري/حساب تاجر) — وتؤجل التسجيل الرسمي لحد ما تتأكد المنتج شغال ومطلوب. لما تكون جاهز تفتح التسجيل للجميع، رجّع `INVITE_ONLY=false`.

## رفعه على Replit

1. اعمل مشروع Node.js جديد على Replit (أو استورد المجلد ده كـ zip / من GitHub)
2. من تبويب **Secrets** (مش ملف .env) ضيف على الأقل:
   - `JWT_SECRET` = أي نص عشوائي طويل
   - `ANTHROPIC_API_KEY` = مفتاحك من console.anthropic.com
   - `SMTP_HOST` / `SMTP_USER` / `SMTP_PASS` = لو عايز إيميلات حقيقية (اختياري بس مهم قبل الإطلاق)
3. دوس **Run** — Replit هيعمل `npm install` و`npm start` تلقائيًا
4. لما تكون جاهز تنشره على رابط دائم، استخدم تبويب **Deployments**

## هيكل المشروع

```
server.js               نقطة الدخول الرئيسية (+ Helmet + CORS قابل للتقييد)
db.js                    تخزين بسيط بملفات JSON (بديل الإنتاج: Postgres — واجهة القراءة/الكتابة موحّدة عشان تستبدلها بسهولة)
lib/aiExtract.js         استدعاء Anthropic API الحقيقي لقراءة الفاتورة من الصورة
lib/email.js             إرسال بريد حقيقي (SMTP) أو وضع تجريبي لو مفيش إعدادات
lib/tokens.js            توليد توكنات آمنة لتأكيد البريد/إعادة تعيين كلمة المرور
middleware/auth.js       JWT auth middleware
middleware/rateLimiters.js  حدود معدل الطلبات لمنع إساءة الاستخدام
routes/auth.js           تسجيل / دخول / تأكيد بريد / نسيت كلمة المرور / بيانات المستخدم
routes/invoices.js       رفع + قراءة AI + حفظ + سجل + إحصائيات
public/                  الواجهة الأمامية (HTML/CSS/JS عادي، بدون framework)
```

## الـ API Endpoints

| Method | Path | يحتاج تسجيل دخول | الوظيفة |
|---|---|---|---|
| POST | `/api/auth/register` | لا | إنشاء حساب (يبعت رابط تأكيد بريد تلقائيًا) |
| POST | `/api/auth/login` | لا | تسجيل دخول |
| GET | `/api/auth/me` | نعم | بيانات المستخدم الحالي (فيها `email_verified`) |
| PATCH | `/api/auth/me` | نعم | تحديث الاسم/اللغة/العملة |
| POST | `/api/auth/verify-email` | لا | تأكيد البريد عبر التوكن من الإيميل |
| POST | `/api/auth/resend-verification` | نعم | إعادة إرسال رابط التأكيد |
| POST | `/api/auth/forgot-password` | لا | طلب رابط إعادة تعيين (رد عام دايمًا — بدون كشف وجود الحساب) |
| POST | `/api/auth/reset-password` | لا | تعيين كلمة مرور جديدة عبر التوكن |
| POST | `/api/join-requests` | لا | تقديم طلب انضمام (وضع Invite-only) |
| GET | `/api/admin/join-requests` | مفتاح إدارة | قائمة طلبات الانضمام (فلترة بـ `?status=`) |
| POST | `/api/admin/join-requests/:id/approve` | مفتاح إدارة | قبول الطلب وإنشاء الحساب |
| POST | `/api/admin/join-requests/:id/reject` | مفتاح إدارة | رفض الطلب |
| POST | `/api/invoices/scan` | نعم | رفع صورة + قراءة AI (لا يحفظ بعد) |
| POST | `/api/invoices` | نعم | حفظ فاتورة بعد المراجعة |
| GET | `/api/invoices` | نعم | سجل عمليات المستخدم |
| GET | `/api/invoices/:id` | نعم | تفاصيل عملية واحدة |
| DELETE | `/api/invoices/:id` | نعم | حذف عملية |
| GET | `/api/invoices/stats/summary` | نعم | إحصائيات (إجمالي، متوسط، أكثر الموردين) |

## الحماية الأمنية المُفعّلة

- كلمات مرور مشفّرة بـ bcrypt (10 rounds)
- كلمة مرور لازم تكون 8 أحرف على الأقل + حرف ورقم
- JWT بصلاحية 30 يوم، وتحذير تلقائي في الـ console لو `JWT_SECRET` ضعيف في بيئة `production`
- Rate limiting: 10 محاولات كل 15 دقيقة على تسجيل الدخول/الحساب، 5 كل ساعة على نسيت كلمة المرور، 20 كل دقيقة على قراءة AI
- توكنات تأكيد البريد/إعادة التعيين: عشوائية بـ 256-bit، مخزّنة كـ hash فقط (مش نص صريح)، صالحة لمدة محدودة (24 ساعة / ساعة)، وتُستهلك مرة واحدة فقط
- `forgot-password` بيرجع نفس الرد بالظبط سواء البريد موجود أو لأ (منع اكتشاف الحسابات المسجلة)
- Helmet مفعّل لـ HTTP security headers
- عزل بيانات كامل بين المستخدمين (مختبر) — كل مستخدم يشوف عملياته بس

## اللي لسه ناقص (متعمد — يحتاج قرارات/حسابات خارجية منك)

- قاعدة بيانات إنتاجية حقيقية (Postgres) بدل ملفات JSON
- حدود الاستخدام الفعلية حسب خطة الاشتراك (Free/Pro/Business)
- الدفع (Paymob/Fawry) — يحتاج حساب تاجر رسمي
- واتساب بزنس API — يحتاج توثيق Meta
- ربط الفاتورة الإلكترونية المصرية (ETA) — إجراء رسمي مع مصلحة الضرائب
- تعدد اللغات الكامل في الواجهة (العربية موجودة، الإنجليزية محتاجة تفعيل)
